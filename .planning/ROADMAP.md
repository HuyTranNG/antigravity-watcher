# Roadmap: Antigravity Watcher

## Phase 1: Foundation & Communication (Core Logic)
- **Goal**: Establish the base extension structure and connectivity to Antigravity.
- [ ] P1-T1: Project setup and core activation logic.
- [ ] P1-T2: Implement `Reactor` for Antigravity API communication.
- [ ] P1-T3: Basic data fetch and display in the Output channel.

## Phase 2: Discovery & Automation (Hunter)
- **Goal**: Implement zero-config discovery of Antigravity processes.
- [ ] P2-T1: Develop `Hunter` for process scanning.
- [ ] P2-T2: Support for multi-process/multi-account detection.
- [ ] P2-T3: Integration of Hunter with Reactor.

## Phase 3: Status Bar & Basic Monitoring
- **Goal**: Provide visible, real-time feedback in the Status Bar.
- [ ] P3-T1: Implement Status Bar item with real-time updates.
- [ ] P3-T2: Color-coded health state logic (Healthy, Warning, Low, Critical).
- [ ] P3-T3: Manual refresh command.

## Phase 4: Activity Bar & Rich Visualizations (Dashboard)
- **Goal**: Create a detailed monitoring dashboard in the Activity Bar.
- [ ] P4-T1: Implement Webview dashboard provider.
- [ ] P4-T2: Develop the dashboard UI (charts, reset countdowns).
- [ ] P4-T3: Bi-directional communication between extension and Webview.

## Phase 5: Polishing & Performance
- **Goal**: Finalize settings and optimize background tasks.
- [ ] P5-T1: Implement configuration settings (thresholds, intervals).
- [ ] P5-T2: Optimize background refresh and memory usage.
- [ ] P5-T3: Error handling and troubleshooting messages.
