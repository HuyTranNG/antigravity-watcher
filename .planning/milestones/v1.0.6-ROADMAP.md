# Milestone Archive: v1.0.6 — Auto-Retry & Resilience

**Archived:** 2026-04-12
**Status:** Completed

## Summary
Implement a robust auto-retry mechanism with exponential backoff and UI progress indicators for server congestion errors.

## Accomplishments
- **Reliable Error Detection**: Extension identifies "High Traffic" server congestion via 429 status codes and response body text matching.
- **Exponential Backoff Core**: Implemented retry mechanism with 5 attempts and increasing delays (2s, 4s, 8s, 16s, 32s).
- **Real-time UI Feedback**: Added "Retrying..." status indicators, countdown timers, and attempt counters to Status Bar and Sidebar Webview.
- **Resilience Controls**: Introduced `antigravity-watcher.enableRetry` setting for toggling auto-retry behavior.
- **Quick-Access Toggles**: Added dedicated **AR** icon in the status bar and a switch in the sidebar header.
- **Full CLI Support**: Standalone tool supports `--no-retry` flag.
